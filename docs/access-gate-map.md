# 역할별 접근/로그인 유도 매핑

기준:
- 앱 현재 구현 코드 기준 (`2026-02-26`)
- 로그인 유도는 `requireAuth()` 호출 시 인증 화면(`AuthFlowScreen`) 오픈

## 페이지별 차이

| 페이지 | 비로그인 사용자 | 로그인 멤버 |
|---|---|---|
| 홈 | 소식/책이야기 목록 조회 가능, 구독/좋아요 액션은 로그인 유도 | 전체 기능 사용 가능 |
| 모임 | 추천/검색은 비로그인용 목록(폴백) 표시, 모임 진입/신청/생성은 로그인 유도 | 모임 추천 API, 진입/신청/생성 가능 |
| 책이야기 | 전체 피드 조회 가능, 좋아요/구독/작성/댓글 액션은 로그인 유도 | 전체 액션 가능 |
| 소식 | 조회 중심 기능 사용 가능 | 조회 기능 동일 |
| 마이페이지 | 탭 진입 자체가 로그인 유도(차단) | 전체 기능 사용 가능 |

## 로그인 화면으로 연결되는 버튼/액션

### 전역
- 하단 탭 `마이페이지` 클릭 (비로그인 시)

### 홈
- 사용자 추천 `구독`
- 책이야기 카드 `좋아요`
- 책이야기 카드 `구독`
- 로그인/회원가입 테스트 버튼

### 모임
- `+ 모임 생성하기`
- 내 모임 카드 진입
- 추천/검색 카드 `방문`
- 추천/검색 카드 `가입 신청` 펼치기/제출

### 책이야기
- 글쓰기 FAB
- 추천 사용자 `구독`
- 피드 카드 `좋아요`
- 피드 카드 `구독`
- 댓글 작성/수정/삭제
- 신고 제출

### 헤더
- 알림 아이콘
- 검색 상세의 책 `내 서재(좋아요)` 토글

## 이번 수정 포인트

- `마이페이지` 탭 비로그인 차단 + 로그인 화면 연결
- 모임 화면 비로그인 상태에서 추천 API 호출 제거(폴백 목록 사용)
- 모임 카드 `방문` 버튼도 로그인 게이트 연결
